Рабочие процессы

Существет несколько методологий использовать git для разработки. Такие методологии назвываются рабочими процессами. Почитать про самые известные процессы можно в следующих статьях: - Сравнение процессов [Rus]; - Централизованный рабочий процесс (trunk flow) [Rus]; - Рабочий процесс с функциональными ветками (Feature-branch) [Rus]; - Рабочий процесс Gitflow Workflow [Rus]; - Рабочий процесс Forking Workflow [Rus];

После прохождения всех материалы выше в первый раз, лучше всего сделать перерыв от дальнейшего изучения, и постраться получить по больше практики в использовании git на реальных проектах.
